We grow trust, not just models.

A self-improving eval harness: it watches an agent fail, mints a check to catch that failure forever, and proves it stayed honest.

The problem

Evals are the bottleneck on safe autonomy. They’re written by hand, they go stale, and when a system grows its own evals, nobody can tell if it improved or just learned to flatter its own grader.

How it works

  1. Catch a failure — a claim with no real support in its cited source.
  2. Cluster it with similar failures by meaning — Atlas vector search.
  3. Mint one general check that catches the whole class.
  4. Grow & prune — the rubric improves itself.

The rubric is the harness. Each branch of the bonsai is one check.

The words, simply

Agent‑under‑test
the AI you want to trust (today: a Gemini 3.5 cited‑answer agent).
Check
one rule the output must pass.
Rubric / harness
the growing set of checks.
Mint
create a new general check from a cluster of failures.
Gold set
a frozen answer key the harness is graded against — that the loop can’t read, so it can’t cheat.

Why it matters — and where it goes

A domain expert defines what “good” means; Bonsai turns it into a living harness that matures itself — instead of a spec that rots in a doc.

Today it’s cited answers. Same loop, any agent: a video or image generator, a contract analyzer, a support bot. The mechanism doesn’t change — only the contract does.

The honesty rail

Anyone can turn a green light on. Here’s why you can’t fake this one: the loop that writes the checks has no code path to the answer key it’s graded against — a test fails the build if they ever touch. We report direction + a confidence interval, never a bare %.
