We grow trust, not just models.

A self-improving eval harness: it watches an AI agent fail, mints a brand-new check to catch that failure forever, grows and prunes its own rubric — and proves it stayed honest while doing it.

Evals are the bottleneck on safe autonomy — not model capability, trust. Bonsai makes the eval harness something that grows itself, and a domain expert can actually own.

The problem

Every team shipping an AI agent needs evals — the checks that verify the agent actually does its job. But the trust layer is broken three ways at once: evals are written by hand, they go stale as the agent and the world change, and when a system grows its own evals nobody can tell whether it improved or just learned to flatter its own grader. You can’t ship an agent you can’t continuously verify.

What Bonsai does

  1. Catch it runs your agent and catches a failure — here, a claim with no verbatim supporting quote in the source it cites.
  2. Cluster it groups that failure with similar ones by meaning (MongoDB Atlas vector search), not by keyword.
  3. Mint it writes a brand-new general check that catches the whole class — kept only if it passes every known-good answer and catches ≥2 distinct siblings.
  4. Grow & prune the rubric grows and trims itself, automatically, over the agent’s real outputs.

The growing rubric is the eval harness. On the demo, each branch of the bonsai is one check it can now run — click a branch to trace the cluster of failures it was minted from.

The words, simply

Agent‑under‑test
the AI you want to trust — here a Gemini 3.5 cited‑answer agent, but it could be a video generator, an image model, or a support bot.
Check
one rule the output must pass — “every number must appear in a cited source.”
Rubric / harness
the growing set of checks — the thing that verifies your agent.
Mint
create a new general check from a cluster of real failures.
Gold set
a small, frozen, human‑authored answer key the harness is graded against — and that the improving loop has no way to read, so it can’t cheat.

Why it matters — and where it goes

A domain contract owner — the expert who actually knows what “correct” means — defines the contract. Bonsai turns that into a living harness that matures as the agent and the domain evolve, instead of a static spec that rots in a doc. It simplifies the hardest part of shipping AI: building and maturing the evals.

Today the contract is “cited answers.” The same loop owns the contract for any domain agent — a video generator (does the clip match the brief? any artifacts? on‑brand?), an image model, a contract analyzer, a medical‑summary bot. The mechanism doesn’t change — only the contract does.

The honesty rail

Anyone can make a green light turn on. Here’s why you can’t fake this one: the loop that writes the checks has no code path to the answer key it’s graded against — separate files, and a test fails the build the instant they touch. We report improvement as a direction + a confidence interval, never a bare percentage. That honesty is the product.
